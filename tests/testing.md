# Тестирование и проверка работоспособности

## Метод тестирования
Использовано сценарное тестирование (tabletop testing), при котором
каждый сценарий атаки был разобран пошагово.

## Тест 1: Фишинговая атака
Пароль пользователя был скомпрометирован.
Результат: пароль сменен, активные сессии завершены.

## Тест 2: Вредоносное ПО
Запущена антивирусная проверка.
Результат: вредоносное ПО обнаружено и удалено.

## Тест 3: DDoS-атака
Смоделирована перегрузка сервиса.
Результат: трафик ограничен, сервис восстановлен.

## Тест 4: Компрометация аккаунта
Обнаружен вход с неизвестного IP.
Результат: доступ восстановлен, настройки защищены.

## Вывод
Все разработанные сценарии реагирования являются логичными,
последовательными и могут быть применены на практике.
