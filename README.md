# Кризисный план действий при кибератаках

## Цель проекта
Создание пошагового плана реагирования на кибератаки для минимизации
ущерба и быстрого восстановления работы систем.

## Актуальность
При отсутствии четких инструкций действия во время кибератаки
становятся хаотичными, что приводит к потере данных и времени.

## Типы инцидентов
- Фишинг
- Вредоносное ПО
- DDoS-атаки
- Утечка данных
- Компрометация учетных записей

## Этапы реагирования
1. Обнаружение
2. Оценка
3. Локализация
4. Устранение
5. Восстановление
6. Анализ инцидента

## Проверка работоспособности
План протестирован с помощью сценарного (tabletop) тестирования.

## Как использовать
Выберите тип атаки в папке docs и следуйте инструкциям.
